Product spec

  • Dashboard home with historical archive, recent-first sorting, and pinned URL-driven filters.
  • Separate competitor reports for GitHub Copilot, Cursor / Bugbot, Qodo, Greptile, Gemini Code Assist, Claude Code / Anthropic, and Cubic.
  • Historical preservation model where one sweep run inserts multiple competitor-specific reports instead of a merged report.
  • Detail page per competitor report with finding log, validation state, CodeRabbit comparison, and Level 1 / 2 / 3 source ledger.
  • History browsing by date preset, archive band, custom range, and run ID.
  • Artifact download support for source PDF and PNG previews when attached.

Recommended architecture

  • Frontend: Flask-rendered analytics UI with Bootstrap and a compact dashboard stylesheet.
  • Backend/API: Flask blueprint plus JSON endpoints for reports, report detail, findings, and sources.
  • Storage: SQLite for preserved metadata and file storage for PDF/PNG artifacts.
  • Ingestion: JSON payload per sweep run that inserts one run record and multiple competitor report records.

How ingestion works

  1. A sweep produces one JSON payload with one run record plus multiple competitor-specific reports.
  2. scripts/ingest_competitor_sweep.py validates the payload and inserts the run without overwriting history.
  3. Each report stores the report window used, latest changelog state, validation coverage, findings, sources, artifacts, tags, and flags.
  4. The dashboard reads preserved records directly from SQLite and exposes them through server-rendered pages and JSON endpoints.

The seeded data in this prototype is illustrative sample data for validating the archive UX and ingestion model.
